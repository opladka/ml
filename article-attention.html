Attention Is All You Need: A Deep Dive into Transformer Architecture

How a 2017 paper from Google Brain quietly dismantled thirty years of sequential thinking in neural networks — and why that matters for everything from search to protein folding.

AR
Arjun Rao
June 3, 2026
14 min read
§

Before 2017, the dominant paradigm for sequence modelling was the recurrent neural network. RNNs — and their more capable cousins, LSTMs and GRUs — processed sequences one token at a time, maintaining a hidden state that tried to compress everything seen so far into a fixed-size vector. It worked. Up to a point.

The fundamental limitation was gradient flow. In a sequence of 512 tokens, the gradient signal from the last token has to travel backwards through 512 multiplicative operations to reach the first. By the time it arrives, it has either vanished or exploded. Long-range dependencies — like resolving a pronoun three paragraphs after its antecedent — were notoriously difficult to learn.

"The dominant sequence transduction models are based on complex recurrent or convolutional neural networks. We propose a new simple network architecture, the Transformer, based solely on attention mechanisms." — Vaswani et al., 2017

The Transformer discarded recurrence entirely. Instead, every token in a sequence attends directly to every other token in a single operation. The path length between any two positions is constant: O(1) rather than O(n). This simple change had consequences that are still being felt today.

The Anatomy of Attention

The core mechanism is scaled dot-product attention. Given a sequence of input vectors, you project each one three times: into a query (Q), a key (K), and a value (V). The output for each position is a weighted sum of all the values, where the weight is determined by the compatibility between that position's query and every other position's key.

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V

The √dₖ scaling factor prevents the dot products from growing so large that they push the softmax into regions of extremely small gradients. Without it, training becomes unstable in high dimensions.

Multi-Head Attention

A single attention head learns one pattern of compatibility. Multi-head attention runs h attention heads in parallel, each with its own learned projections, then concatenates and projects the results. Each head can specialise: one might track syntactic agreement, another coreference, another positional proximity.

def multi_head_attention(Q, K, V, h, d_model):
    d_k = d_model // h
    heads = []
    for i in range(h):
        Q_i = Q @ W_q[i]   # (seq, d_k)
        K_i = K @ W_k[i]
        V_i = V @ W_v[i]
        heads.append(attention(Q_i, K_i, V_i))
    return concat(heads) @ W_o  # project back to d_model

Positional Encoding

Attention is inherently permutation-invariant: shuffle the tokens and you get the same attention weights (just reordered). That's a problem for language, where word order is everything. The Transformer injects positional information by adding a sinusoidal encoding to each token embedding before the first layer.

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

The sinusoidal design was chosen so that relative positions could be represented as a linear function of the encodings — allowing the model to generalise to sequences longer than those seen during training. Modern models typically replace this with learned positional embeddings or rotary position encodings (RoPE).

The Full Encoder-Decoder Stack

The original Transformer was designed for sequence-to-sequence tasks like machine translation. The encoder processes the source sequence into contextualised representations. The decoder generates the target sequence autoregressively, attending both to its own previous outputs and to the encoder's output via cross-attention.

Component Inputs Key operation
Encoder self-attention Source tokens Each token attends to all source tokens
Decoder self-attention Target tokens (masked) Each token attends to previous target tokens only
Cross-attention Decoder + Encoder output Target tokens attend to all source representations
Feed-forward Attention output Two linear layers with ReLU, applied position-wise

Each sub-layer is wrapped in a residual connection followed by layer normalisation. This is the "Add & Norm" pattern you'll see in every Transformer diagram. The residual connections are critical: they provide a gradient highway that bypasses each sub-layer and makes training deep stacks feasible.

Why It Generalised So Far

It's worth pausing to ask why a machine translation architecture became the foundation of everything from GPT to AlphaFold. The answer is that the Transformer's inductive biases are remarkably weak. Unlike CNNs (which assume spatial locality) or RNNs (which assume temporal ordering), the Transformer makes no strong assumption about the structure of its input. It will happily learn whatever patterns exist in the training data — syntactic, semantic, structural, or otherwise.

This generality, combined with the parallelism that makes large-scale training feasible, is why BERT (encoder-only), GPT (decoder-only), and T5 (encoder-decoder) all converged on the same core architecture. The innovations since 2017 — flash attention, rotary embeddings, grouped query attention, mixture of experts — are optimisations on top of a structure that has remained essentially unchanged.

"We are still largely running the 2017 architecture. We've made it much bigger, much faster, and much cheaper — but the core attention mechanism is the same eight lines of matrix algebra." — unnamed researcher, NeurIPS 2024 panel

Implementation Notes

If you're implementing a Transformer from scratch, a few things trip people up that the paper glosses over. First, the learning rate schedule matters enormously — the original paper uses a warmup followed by inverse square root decay, and deviating from this often causes divergence. Second, weight tying between the embedding and the final linear layer is standard in language models and saves a significant fraction of parameters. Third, the feed-forward dimension is typically 4× the model dimension, not the same size.

class TransformerBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_ff, dropout=0.1):
        self.attn = MultiHeadAttention(d_model, n_heads)
        self.ff   = FeedForward(d_model, d_ff)
        self.ln1  = nn.LayerNorm(d_model)
        self.ln2  = nn.LayerNorm(d_model)
        self.drop = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        x = x + self.drop(self.attn(self.ln1(x), mask))  # pre-norm
        x = x + self.drop(self.ff(self.ln2(x)))
        return x

Note the pre-norm variant shown above — applying layer norm before each sub-layer rather than after. Most modern implementations (GPT-2 onward) use pre-norm because it stabilises training at large scale, even though the original paper used post-norm.
