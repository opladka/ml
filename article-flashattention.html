FlashAttention 3: What Changed and Why It Matters

DP
Dev Patel
May 12, 2026
7 min read
Tutorials

As context lengths stretch from 8K tokens to over a million, computing self-attention becomes the primary computational bottleneck in LLM execution. While float execution speeds (FLOPs) continue to climb on modern accelerators, reading data to and from memory remains remarkably slow. Attention is **memory-bandwidth bound**.

Standard self-attention computes and writes the full N x N attention matrix to High Bandwidth Memory (HBM), reads it back for softmax scaling, and writes the output back to HBM. FlashAttention tiling bypasses this cycle completely.

SRAM Tiling and Cache Blocking

FlashAttention divides the Query, Key, and Value matrices into block blocks. Instead of materializing the intermediate matrix in slow HBM, blocks are loaded to fast GPU **SRAM (Static RAM)** on-chip, scaled using running softmax coefficients, and written out to the final output matrix directly.

What's New in FlashAttention 3

FlashAttention 3 leverages the latest Hopper GPU architecture updates. Key enhancements include:

  • Warp-Group GEMM: Utilizing Tensor Core matrix multiply operations asynchronously.
  • Overlapping Compute and Memory: Pipelining block load steps from HBM to SRAM while computing the previous block on Tensor Cores concurrently.
  • Low Precision Scale: Supporting FP8 formats with dynamically scaled mantissas.
flash_attn.cu (CUDA Kernel Concept) 
// CUDA Block Tiling Loop
__global__ void flash_attention_kernel(float* Q, float* K, float* V, float* O) {
    int tx = threadIdx.x;
    int bx = blockIdx.x;

    // Load Q block to Shared Memory (SRAM)
    __shared__ float Q_shared[BLOCK_SIZE][D_K];
    __shared__ float K_shared[BLOCK_SIZE][D_K];
    
    // Async load from HBM
    load_async(Q_shared, &Q[bx * BLOCK_SIZE], tx);
    
    for (int j = 0; j < num_blocks; j++) {
        // Overlap K, V load with GEMM compute
        load_async(K_shared, &K[j * BLOCK_SIZE], tx);
        __syncthreads();
        
        // Accumulate attention scores locally on-chip
        compute_block_attention(Q_shared, K_shared);
    }
}
