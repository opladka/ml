Mixture of Experts: Scaling Without the Cost

SK
Sarah K.
May 18, 2026
8 min read
Research

As LLMs scale, training denser parameter sets becomes financially and computationally prohibitive. Scaling parameters improves capacity, but standard compute demands multiply linear costs with every layer extension. Mixture of Experts (MoE) addresses this tradeoff by scaling model parameters while keeping computational costs constant.

In a standard Feed-Forward Network (FFN), every token passes through the same dense matrix layers. In an MoE layer, the single dense FFN is replaced by multiple independent "expert" FFNs. A lightweight gating network, or **router**, directs each token to only one or two experts.

Token Routing Architecture

For every token vector x, the router computes a probability distribution over the N experts:

G(x) = Softmax( KeepTopK( H(x), K ) )

Where H(x) is a raw linear projection of the token vector x, and the KeepTopK operator sets all values to negative infinity except for the top-k highest scoring elements.

Load Balancing and Routing Failure

If the router is left to optimize without constraints, it often collapses into a state where only a few experts receive most of the traffic. This defeats the purpose of distributed experts and creates bottlenecks.

To prevent this, models use an auxiliary **load balancing loss** during training. This encourages equal token distribution across all experts:

MoERouter.py (PyTorch Implementation) 
import torch
import torch.nn as nn
import torch.nn.functional as F

class Top2Router(nn.Module):
    def __init__(self, d_model, num_experts):
        super().__init__()
        self.wg = nn.Linear(d_model, num_experts, bias=False)

    def forward(self, x):
        # x: [Batch, SeqLen, D_model]
        logits = self.wg(x) # [Batch*SeqLen, num_experts]
        
        # Softmax over expert scores
        probs = F.softmax(logits, dim=-1)
        
        # Get top-2 experts
        top_val, top_idx = torch.topk(probs, k=2, dim=-1)
        return top_idx, top_val
