RLHF Demystified: How Language Models Learn Human Preferences

PM
Priya Menon
May 28, 2026
9 min read
Tutorials

Pre-training learns word distributions across web text. Instruction tuning alignments refine standard output layouts. However, aligning output to match complex human values (safety, tone, helpfulness, and nuance) requires explicit training reinforcement. This is where Reinforcement Learning from Human Feedback (RLHF) comes in.

RLHF bridges the gap between next-token predictions and user expectations. It decomposes this problem into three stages: supervised fine-tuning (SFT), reward model training via comparative rankings, and fine-tuning using policy reinforcement (PPO).

The Three Stages of RLHF

  1. Supervised Fine-Tuning (SFT): The base model is trained on curated instructions and responses written by human educators.
  2. Reward Modeling: The model generates multiple candidate responses for a given prompt. Humans rank these outputs. A reward model (RM) is then trained to output a scalar score predicting the preferred human ranking.
  3. PPO Fine-Tuning: The SFT model policy is optimized against the reward model score using Proximal Policy Optimization. A KL divergence penalty prevents the policy from drifting too far from the SFT base weights.

The Loss Function of Reward Models

A reward model is trained by taking a preferred output yw and a dispreferred output yl for a prompt x. The goal is to maximize the difference between their scalar scores:

Loss(θ) = - E(x, yw, yl) [ log( σ( rθ(x, yw) - rθ(x, yl) ) ) ]

Where rθ represents the reward model scores and σ is the sigmoid function. By mapping this differential log sigmoid, preferred responses are pushed toward higher scores while dispreferred drafts are suppressed.

RewardLoss.py (PyTorch Implementation) 
import torch
import torch.nn as nn

class RewardModelLoss(nn.Module):
    def __init__(self):
        super().__init__()

    def forward(self, chosen_rewards, rejected_rewards):
        # chosen_rewards: [Batch] scalar outputs for preferred drafts
        # rejected_rewards: [Batch] scalar outputs for rejected drafts
        loss = -torch.log(torch.sigmoid(chosen_rewards - rejected_rewards)).mean()
        return loss
